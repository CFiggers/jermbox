# Guidelines for Contributing

## Reporting bugs

* Check closed and open issues to see if your problem has been run into before or already solved.
* If you can't find an open issue or solution for your problem, you should open a new issue. If
  there is a closed issue that is relevant, make
  sure to reference it.
* Include a comprehensive description of the problem and instructions on how to reproduce it.

## Contributing Changes

If you want to contribute code to the project, please submit a pull request and follow the
guidelines below.

* Include a description of the changes made.
* Be consistent with style

## Suggesting Changes

To suggest changes, open an issue on GitHub. Check GitHub for other issues
that may be related to your issue before opening a new suggestion. Include a good description
of the problem that is being solved and potential solutions.
